We snap thousands of photos. Most of them never get seen again. We tell incredible stories — once — to a friend, then forget the detail. The trip ends, the world dissolves back into a folder named 2026/SG.

  • Photos pile up in chronological folders

  • Cultural context lives only in the moment
  • Travel ends → memories fade

🌍 Planet B-612

  • Each photo becomes a living 3D model on your planet
  • An AI fox identifies it and tells you the deep local context
  • A codex grows; a planet evolves; a fox writes diary entries about your finds
  • Years later, you can still walk through the world you brought back

Architecture
AGENT HIERARCHY & FLOW

One router. Two pipelines. One autonomous fox.

A single Orchestrator sits between the user and every specialist. It dispatches each input into the right pipeline — identification or planning — while Fox Life runs in parallel as its own autonomous loop.
Tier 0 · Entry
👤
User · text / photo / tap
Tier 1 · Router
🧠
Orchestrator
Classifies intent (message / photo / task / landmark) and dispatches to one of two pipelines. Owns shared planet context.
Tier 2 · Specialists
👁️
Vision · photo pipeline
Qwen-VL-Max → 1 of 20 categories + structured details. Receives existing codex for de-dupe.
📖
Story · enriches Vision
Called by Orchestrator after Vision succeeds. Generates deep historical / cultural Insight in the fox's voice.
✈️
Planner · text pipeline
SG local-guide brain. Builds 30-90-min quests grounded in time, weather, festivals, hawker rhythms — and asks Sponsor.
🎁
Sponsor · sub-call of Planner
When keywords match (SuperAI / Tiger / Mr. Coconut), Planner injects a clearly-badged sponsored quest into its result.
🦊
Fox Life
Runs in its own timer-driven loop, independent of the Orchestrator. Picks activities based on bond level + current planet state.

Problems & Solutions
PROBLEMS WE HIT · HOW WE FIXED THEM

Six hard problems. Six shipped fixes.

🐛
Laksa → "landmark"
Problem: Vision mis-classified hawker food as buildings, then UI silently showed fake fallback data.
Fix: Negative-constraint prompt with food examples; on failure show "AI couldn't read this" + Retry / Manual buttons — never fake.
⏱️
2-minute API hangs
Problem: Server logs showed Qwen calls aborting at ~30 ms — wrong req.on('close') handler killed every request.
Fix: Swapped to res.on('close'), added 50 s server timeout + 60 s client timeout + true Cancel button.
🌍
Models stacking on each other
Problem: New items spawned on top of existing landmarks — a pile, not a planet.
Fix: Spherical spiral search using great-circle distance picks the nearest free spot to each anchor — guaranteed non-overlap.
🧭
Buildings tilted on the sphere
Problem: Models leaned at random Y-angles after placement; flagpoles aimed anywhere.
Fix: Build an orthonormal basis from each spot (radial-up / world-north / east) → quaternion. Models stand upright everywhere, even at poles.
♻️
"Kaya Toast" ≠ "咖椰吐司"
Problem: AI returned slightly different names for the same dish → duplicates in the codex.
Fix: Pass existing codex into the Vision prompt + fuzzy normalize (strip punctuation, common prefixes, lowercase) before dedupe.
✨
Highlight hidden by modal
Problem: Clicking a codex card highlighted the model, but the full-screen detail modal covered the planet — invisible feedback.
Fix: Right-docked the modal, lowered overlay opacity, added halo + floating arrow mesh that survives on top.
